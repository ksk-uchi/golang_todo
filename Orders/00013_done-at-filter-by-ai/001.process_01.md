# 作業手順書: 完了日時に対する自由文でのフィルタを可能にする

## 目的

`Orders/00013_done-at-filter-by-ai/001.order.md` の要求に基づき、Go バックエンドにおいて自然言語でToDoの完了日時をフィルタする機能を実装する。AI クライアントには `go-genai` を使用する。

## 注意事項 (サブエージェント向け)

- まず `Orders/00013_done-at-filter-by-ai/001.order.md` を読み、要件と制約を全体的に把握すること。
- プロジェクトの既存ルール (Google Wire を用いた DI、Ent を用いたデータベース操作) に準拠すること。
- 対象ファイル以外 (`Orders/` 下のファイルなど) は `git add` やコミットに含めないこと。

## 手順

### Step 1: リポジトリ側の関数・Tool 定義の実装

1. `backend/repositories/todo.go` (または関連ファイル) に、完了日時を範囲指定でフィルタし ToDo 一覧を取得する関数 (例: `FetchTodosByDoneAt`) を実装する。
   - **重要**: セキュリティ要件のため、必ずコンテキストなどから取得できるユーザーID (`user_id`) でもフィルタすること。
   - 実装の際は既存コードを参考に、`done_from` と `done_to` (`time.Time` またはそれに相当) を許容する仕様にする。
2. `backend/function_declarations/list_todos_by_done_at.go` を作成する。
   - `go-genai` 用の Function Declaration (`Tool` に渡すためのスキーマ定義。`done_from` と `done_to` をプロパティとして持つ) を定義する。
   - ここには、AIから指定された引数を受け取り、実際に `backend/repositories/todo.go` のDBアクセス関数（Step 1-1で作成したもの）を呼び出して `[]*ent.Todo` を返す関数 `ListTodosByDoneAt` も実装する。

### Step 2: AIサービスの実装 (`backend/services/ai.go`)

1. **クライアント生成・インターフェース化**:
   - 環境変数 `GOOGLE_API_KEY` を使用し、AI クライアントを生成・提供する `CreateAIClient` 関数を実装・エクスポートする。
   - モックテストを可能にするため、クライアントを直接使うのではなくインターフェースに基づく呼び出しにするか、通信をモックできる設計にすること。
2. **関数 `DecideFilterTodosFunction` の実装**:
   - 引数: `ctx context.Context, query string`
   - 処理フロー:
     1. 関数内部で `CreateAIClient` を呼び出し、`client` を作成する。
     2. `client` 経由でクエリを AI に投げる。プロンプトには「現在の時刻は〜」および「対応できる Tool が無い場合は…」の制約と、ユーザが入力した `query` を含める。
     3. リクエスト時には Tool として `ListTodosByDoneAt` の定義を指定する (モデル: `gemini-3-flash-preview`)。
     4. AIから 関数名と引数のセットが返ってきたら、それを DTO に詰めて return する。Tool Call が発生しなければ nil を返す。
3. **関数 `FilterTodos` の実装**:
   - 引数: `ctx context.Context, function_name string, args interface{}`
   - 処理フロー:
     1. 呼び出し元から渡された関数名 (`function_name`) をチェックする。
     2. `function_name` が `ListTodosByDoneAt` なら、`backend/function_declarations` パッケージに実装した `ListTodosByDoneAt` を呼び出す。
     3. 関数から返ってきた `[]*ent.Todo` をそのまま戻り値として返却する。

### Step 3: テストの実装

1. `backend/services/ai.go` および `backend/function_declarations/list_todos_by_done_at.go` のテスト (`ai_test.go` など) を実装する。
2. モックを利用して実際に Gemini API へリクエストが飛ばないようにする。
   - Tool が選択される正常系や、該当する Tool がない場合のシナリオなど、複数のケースを網羅すること。
3. バックエンドの全テストを実行 (`go test ./...` など) し、全てパスすることを確認する。

> **コミット作成ポイント【2】**
