# 完了日時に対する自由文でのフィルタを可能にする

## 背景

- `todos` テーブルには `done_at` という完了日時を保持するカラムがある

## 要件

- 「直近一ヶ月の間に完了になったタスクを表示」といった完了日時に対する自由文でのフィルタを可能にする

## 対応内容

1. `backend/cmds/check_genai/main.go` と `backend/cmds/check_genai/function_declerations/list_todo_by_condition.go` を PoC として `backend/services/ai.go` と `backend/function_declarations/list_todos_by_done_at.go` に `go-genai` を使った AI に tool を指定した状態でリクエストを送信することが出来る実装を行う
   - API Key は環境変数 `GOOGLE_API_KEY` で指定する
     - 環境変数 `GOOGLE_API_KEY` は存在するものとして実装して良い
   - 今回 tool に追加する関数
     - 名前: `ListTodosByDoneAt`
     - 説明: ToDo の完了日時に対して範囲指定で ToDo 検索する
     - 引数
       - `done_from`: 完了日時の開始範囲 (RFC3339形式)
       - `done_to`: 完了日時の終了範囲 (RFC3339形式)
     - 戻り値
       - ToDo 一覧 `[]*ent.Todo`
     - 実装
       - `backend/repositories/todo.go` に `done_at` を範囲指定でフィルタし、ToDo 一覧を取得する関数を実装し、それを呼び出す
         - 他者の ToDo を参照できないように、その関数はユーザー ID でもフィルタすること
   - `backend/services/ai.go`
     - AI に接続するためのクライアントの作成などは同ファイルに `CreateAIClient` を実装しそれを用いる
     - todo をフィルタするための関数 `DecideFilterTodosFunction` を実装する
       - 引数
         - `ctx`: context.Context
         - `query`: 自由文でのフィルタクエリ
       - 戻り値
         - 実行する関数名と引数のセットを DTO に詰めて返却する
           - nil になる可能性がある
       - 下記２つの要素は必ず AI へのリクエストに含めるようにする
         - `{Text: time.Now().Format("現在2006年1月2日15:04:05です。")}`
         - `{Text: "使用できる Tool が無い場合は「対応できる Tool がありません」とだけ回答するようにしてください。"}`
       - tool には `ListTodosByDoneAt` を指定する
     - 実行する関数名と引数のセットを元に実際に ToDo をフィルタする関数 `FilterTodos` を実装する
       - 引数
         - `ctx`: context.Context
         - `function_name`: 実行する関数名
         - `args`: 実行する関数引数
       - 戻り値
         - フィルタされた ToDo 一覧 `[]*ent.Todo`
   - 完了条件
     - [ ] 既存の全テストがパスすること
     - [ ] Git コミットの作成
2. `backend/services/ai.go` と `backend/function_declarations/list_todos_by_done_at.go` のテストを実装する
   - モックを使って AI に対するリクエストが発生しないようにする
   - 完了条件
     - [ ] 全テストがパスすること
     - [ ] Git コミットの作成

## 制約事項

- `AI` に対するリクエストには既にインストールされている `go-genai` ライブラリを使用すること
  - モデルは `gemini-3-flash-preview` を使用すること
