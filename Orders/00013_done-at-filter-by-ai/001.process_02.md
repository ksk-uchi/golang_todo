# 作業手順書: テスト用の Spy リポジトリを testify/mock に置き換えて共通化

## 目的

`Orders/00013_done-at-filter-by-ai/001.review.md` の指摘事項に基づき、複数ファイルに重複して定義されている `todo` リポジトリのテスト用スパイ実装を、`testify/mock` を活用した共通のモックオブジェクトにリファクタリングする。

## 注意事項 (サブエージェント向け)

- 作業開始前に必ず `Orders/00013_done-at-filter-by-ai/001.review.md` の `[Do]` の内容を確認すること。
- テストファイルのみの変更およびテストユーティリティの追加となるため、プロダクトコード（テスト以外）のロジックは変更しないこと。
- `testify/mock` の作法に乗っ取り、厳密な引数および戻り値の型定義を行うこと。
- **1つの指摘事項につき1コミットとするため、本タスク全体で1つのコミットを作成すること。**

## 手順

### Step 1: 共通 Mock オブジェクトの作成

1. `backend/testutils/mock_todo_repo.go` ファイルを作成する（`backend/testutils` ディレクトリが存在しない場合は作成する）。
2. そのファイル内で `github.com/stretchr/testify/mock` をインポートし、`backend/repositories` の `ITodoRepository` インターフェースを実装する `MockTodoRepository` 構造体を作成する。
   - `MockTodoRepository` は `mock.Mock` を埋め込んでいること。
   - `ITodoRepository` が持つすべてのメソッド (`FetchAllTodos`, `FetchTodoByID`, `InsertTodo`, `UpdateTodo`, `DeleteTodo`, `FetchTodosByDoneAt` など) を、`testify/mock` の一般的なインターフェース実装手法 (`m.Called(...)` と `args.Get(0)` など) に従って実装する。

### Step 2: 既存のテストのスパイを書き換え (services)

1. `backend/services/todo_test.go` を開き、ファイル内で定義されている `spyTodoRepo` およびそれを使用している箇所を探す。
2. これらを `backend/testutils` の `MockTodoRepository` を使用するように置換する。
   - 今まで `FetchAllTodosFunc` などの関数ポインタに代入していたモックの振る舞い設定を、`repoMock.On("MethodName", ...).Return(...)` のような `testify/mock` の設定方法に書き換えること。
3. `spyTodoRepo` の構造体定義を削除する。

### Step 3: 既存のテストのスパイを書き換え (function_declarations)

1. `backend/function_declarations/list_todos_by_done_at_test.go` を開き、内部で定義されている独自の `spyTodoRepo` を探す。
2. これも同様に `backend/testutils` の `MockTodoRepository` を使用するように置換する。
   - Step 2 と同様に、関数ポインタへの代入を `On(...).Return(...)` の形に書き換えること。
3. `spyTodoRepo` の構造体定義を削除する。

### Step 4: テストの実行とコミット

1. バックエンドディレクトリ (`backend/`) に移動し、ローカル環境変数を使って全体のテストを実行し、すべて成功することを確認する。
   - コマンド例: `cd backend && APP_ENV=local go test ./...`
2. テストが成功したら、今回の変更分を `git add` し、「1点につき1コミット作成」のルールに従いコミットを作成する。
   - コミットメッセージ例: `refactor: replace spyTodoRepo with testify/mock in testutils`
