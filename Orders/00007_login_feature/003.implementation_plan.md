# 実装計画 - レイアウト構造を用いたログイン画面の縦中央配置

## 概要

マジックナンバー (`calc(100vh - ...)`など) を使用せず、Flexbox の仕組みを利用してログイン画面 (`frontend/src/app/login/page.tsx`) を縦中央に配置します。
そのために、`frontend/src/app/layout.tsx` の構造を変更し、`main` および `div.container` が利用可能な高さを最大限占有するようにします。

## 背景

- 前回提案のマジックナンバーによる高さ指定は、保守性が低いため却下されました。
- `Root Layout` の構造自体を調整し、コンテンツ領域が自然に画面高さを埋めるようにすることで、子コンポーネント (`page.tsx`) 側で `flex-1` や `h-full` を利用して中央揃えが可能になります。

## 仕様詳細

### 対象ファイル

1. **frontend/src/app/layout.tsx**
   - `main` 要素に `flex flex-col` を追加し、子要素 (`container`) が高さを埋められるようにします。
   - `div.container` に `flex-1 flex flex-col` を追加し、`main` 内の余白を全て埋めるようにします。

2. **frontend/src/app/login/page.tsx**
   - ルートの `div` に `w-full flex-1` を追加し、親コンテナ (`div.container`) の内部で最大限広がるようにします。
   - これにより、`items-center justify-center` が有効になり、Card コンポーネントが中央に配置されます。

## 懸念点と確認事項

- **他のページへの影響**: `layout.tsx` は全ページ共通です。Todo リスト画面などのコンテンツが多いページでも問題ないか考慮が必要です。
  - `flex-1` は「最低限の高さ」ではなく「余白を埋める」指定であり、中身が溢れる場合は通常通り伸びるため、スクロール挙動 (`overflow-y-auto` は `main` にある) に悪影響はないと想定されます。

## 手順

1. `frontend/src/app/layout.tsx` を修正。
2. `frontend/src/app/login/page.tsx` を修正。
3. 動作確認。
