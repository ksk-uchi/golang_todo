# ガイドライン: バックエンド DI・リポジトリ リファクタリング

対象の担当者: Backend-DI-Agent

## 目的

主に Google Wire と連動した依存性注入 (DI) の改善、およびリポジトリ層・サービス層の設計改善を行います。
他のエージェントと並行稼働するため、指定された【対象ファイル】を主に変更し、他のエージェントとのコンフリクトを最小限に抑えてください（Wire適用のためコンストラクターでの引数変更による各所の微修正は許容されます）。

## 対象ファイル

- `backend/di/wire.go` （および生成される `wire_gen.go`）
- `backend/repositories/*.go`
- `backend/services/*.go`
- ハンドラーのコンストラクター部分への適用箇所
- 該当するテストファイル

## やるべきこと

1. **リポジトリの DI 設計改善**:
   - リポジトリ内部で `ent.FromContext(ctx)` による DB クライアント取得を廃止します。
   - 代わりに `ent.Client` インスタンスをリポジトリ構造体のプロパティに持たせ、コンストラクター (`NewTodoRepository` 等) 経由で直接注入するように設計を変更してください。
2. **Factory モジュールの廃止**:
   - `TodoHandler` 等でリクエストごとに利用されている `TodoServiceFactory` などのFactory運用を廃止し、直接 `TodoService` を注入してDIコンテナの設定をシンプルにしてください。
3. **Wireの再ビルド**:
   - ディレクトリ内のパス状況を確認しつつ、`wire` コマンドを実行して `wire_gen.go` を最新状態に正常に生成・更新してください。
4. **テストの修正と通過（必須要件）**:
   - DI 設計の変更に伴い、リポジトリ層やサービス層の初期化方針が変わるため、既存のモックテストなどが落ちる可能性があります。
   - 実装作業後、必ず `backend/` ディレクトリに移動して `go test ./...` を実行し、**すべてのテストがパスすることを確認**してから作業を完了してください。
   - **注意: 絶対に `git commit` や `git add` 等のgit操作は実行しないでください。** 修正のコミットや `pre-commit` フックの実行はすべてのエージェントの作業完了後にメインエージェントがまとめて行います。コードの修正とテスト確認のみを行って終了してください。
