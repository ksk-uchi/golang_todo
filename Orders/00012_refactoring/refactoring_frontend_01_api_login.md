# ガイドライン: フロントエンド API・Loginコンポーネント リファクタリング

対象の担当者: Frontend-Auth-Agent

※ `pnpm` を厳守してください（`npm`, `yarn`, `npx` 使用禁止）

## 目的

フロントエンドのAPI通信基盤の改善と、ログイン画面のバリデーション実装を行います。
他のエージェントと並行稼働するため、指定された【対象ファイル】以外は極力変更しないでください。

## 対象ファイル

- `frontend/src/lib/api.ts`
- `frontend/src/app/login/page.tsx`
- 必要に応じて新規のサービス層ファイル (`frontend/src/services/` など)

## やるべきこと

1. **API基盤の改善**:
   - `lib/api.ts` 内の `baseURL: "http://localhost:8080"` などのハードコードを廃止し、環境変数 (`NEXT_PUBLIC_API_URL` 等) を参照するように変更してください。
   - CSRFトークンの取得手段において、`document.cookie` からの文字列操作による直接抽出から、より堅牢で標準的なパーサー等を用いた方法に書き直してください。
2. **ログインコンポーネントの改善とZod導入**:
   - 既存で `zod` がインストールされているため、`login/page.tsx` にて手動で行われている状態管理と手書きバリデーションを、`zod` （可能であれば `react-hook-form` も組み合わせ）を用いたスキーマバリデーションに変更してください。
   - エラーハンドリングに見られる `as Error` 等の型アサーションを排除し、適切な型ガード（`isAxiosError` など）を利用してください。
3. **ビルドとテストの通過（必須要件）**:
   - 実装作業後、`pnpm build` を実行し、**エラーなくビルドできる事を確認**してから作業を完了してください。
   - **注意: 絶対に `git commit` や `git add` 等のgit操作は実行しないでください。** 修正のコミットや `pre-commit` フックの実行はすべてのエージェントの作業完了後にメインエージェントがまとめて行います。コードの修正とテスト確認のみを行って終了してください。
