# サブエージェント向け作業手順書: 検索履歴から最新の5件を表示する API の作成

## 概要

ユーザーが過去の検索結果を再利用できるようにするため、`todo_filter_histories` テーブルから最新5件の履歴を取得する API を実装します。
以下のフェーズに従って順番に実装とテストを進め、完了条件を満たすごとにコミットを作成してください。

## フェーズ1: リポジトリ層の実装

1. **実装対象:** `backend/repositories/todo_filter_history.go` (新規作成)
2. **実装内容:**
   - 対象エンティティの Repository インターフェースおよびその実装を定義します。
   - メソッド `FetchLatestFilters` を実装します。引数で指定された件数 `N` まで、連携された `userID` に紐づく履歴を `created_at` の降順(DESC) で取得してください。
3. **テスト実装:**
   - リポジトリのテストは実装しなくて良いです。
   - ただし今後サービスのテストなどで使用するため、テスト用のモック実装を `backend/testutils/` に新規作成してください。
4. **完了条件:**
   - [ ] 既存の全テストがパスすること (`go test ./...`)
   - [ ] `git add` を用いて変更したファイルのみをステージングし、明確なコミットメッセージでコミットを作成すること

## フェーズ2: サービス層の実装

1. **実装対象:** `backend/services/todo_filter_history.go` (新規作成)
2. **実装内容:**
   - 対象サービスのインターフェースおよびその実装を定義します。フェーズ1の Repository を利用できるようにしてください。
   - `FetchLatestFilters` メソッドを実装します。このメソッド内では、リポジトリ層の同名メソッドを呼び出しますが、呼び出し時の取得件数 `N` は `5` とします。
3. **テスト実装:**
   - フェーズ1で用意したモック実装を用いて、サービスメソッドのテストを実装してください。
4. **完了条件:**
   - [ ] 全テストがパスすること
   - [ ] 変更内容のみをコミットすること

## フェーズ3: ハンドラ層の実装とルーティングの登録

1. **実装対象:** `backend/handlers/todo.go` (または依存関係に応じて適切なファイル)、`backend/routes/todo.go`
2. **実装内容:**
   - フェーズ2のサービスを利用するハンドラメソッド `ListTodoFilterHistories` を実装します。
   - レスポンスは以下の JSON 形式としてください（エラーハンドリング等も適切に行うこと）。
     ```json
     {
       "queries": [
         {
           "query": "一週間以内に完了にしたタスク",
           "id": "019cb6ea-bb06-736c-9358-f3e03223c9e3"
         }
       ]
     }
     ```
   - `backend/routes/todo.go` に `GET /todo/filter_histories` のルーティングを追加します。ログイン中のユーザーIDを用いるため、必ず認証ミドルウェアを通るように設定してください。
     - `backend/routes/router.go` で URL パスの一部 `/todo` は確定しているので注意してください
3. **テスト実装:**
   - エンドポイントへの HTTP リクエストを通じて、レスポンスの JSON フォーマットやステータスコードを検証するテストを実装してください。
4. **完了条件:**
   - [ ] 全テストがパスすること
   - [ ] 変更内容のみをコミットすること
