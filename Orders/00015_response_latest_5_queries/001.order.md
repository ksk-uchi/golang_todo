# 検索履歴から最新の5件を表示する API の作成

## 背景

- ToDo 一覧画面において、ユーザーは過去の検索結果を再利用したい

## 要件

- ユーザーに紐づく過去の検索結果を最大5件まで取得する API を作成する
  - URL: `GET /todo/filter_histories`
  - API のレスポンスは下記のようになるよう実装する
    ```json
    {
      "queries": [
        {
          "query": "一週間以内に完了にしたタスク",
          "id": "019cb6ea-bb06-736c-9358-f3e03223c9e3"
        }
      ]
    }
    ```

## 対応内容

1. `backend/repositories/todo_filter_history.go` を作成し、 `todo_filter_histories` テーブルからユーザーに紐づく最新の N 件の検索条件を取得するメソッドを実装する
   - 関数名は `FetchLatestFilters` とする
   - N は引数で指定できるようにする
   - テスト用のモック実装を `backend/testutils/` に新規作成し実装する
   - 完了条件
     - [ ] 既存の全テストがパスすること
     - [ ] Git コミットの作成
2. `backend/services/todo_filter_history.go` を作成し、 1. で作成したリポジトリメソッドを呼び出すメソッドを実装する
   - 関数名は `FetchLatestFilters` とする
   - N は 5 として呼び出す
   - テストを実装する。
   - 完了条件
     - [ ] 全テストがパスすること
     - [ ] Git コミットの作成
3. `backend/handlers/todo.go` に 2. で作成したサービスメソッドを呼び出すハンドラメソッドを実装する
   - `backend/routes/todo.go` にハンドラメソッドを登録する ( `GET /todo/filter_histories` になるように)
   - 関数名は `ListTodoFilterHistories` とする
   - テストを実装する。
   - 完了条件
     - [ ] 全テストがパスすること
     - [ ] Git コミットの作成
