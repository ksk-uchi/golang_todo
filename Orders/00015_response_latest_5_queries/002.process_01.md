# サブエージェント向け作業手順書: リポジトリの共通処理 (`getUser`, `getClient`) の共通化

## 概要

作成した検索履歴APIおよびTodo系のリポジトリ実装において、コンテキストから DB クライアントやユーザー情報を取得する処理（`getClient`, `getUser`）が重複しているため、これらを `BaseRepository` という共通構造体に切り出し、構成（Composition）によって再利用できるようにリファクタリングを行います。

## フェーズ1: BaseRepository の作成と適用

1. **対象ファイル**
   - `backend/repositories/base.go` （新規作成）
   - `backend/repositories/todo.go`
   - `backend/repositories/todo_filter_history.go`

2. **実装内容**

   **`backend/repositories/base.go` の新規作成:**
   - パッケージを `repositories` とします。
   - `baseRepository` という非公開構造体、または `BaseRepository` という公開構造体を作成し、`client *ent.Client` プロパティを持たせます（名前は既存の規約に合わせて決めてください）。
   - `todo.go` などにある `getClient(ctx)` および `getUser(ctx)` と同等のメソッドを、この `BaseRepository` に生やす形へ移植してください。

   **`backend/repositories/todo.go` の修正:**
   - 構造体の定義において、`client *ent.Client` プロパティを取り除き、代わりに `base *BaseRepository` （または `base BaseRepository`） を持たせてください。
   - コンストラクタ（`NewTodoRepository`）を修正し、`BaseRepository` を初期化して持たせるようにしてください。
   - ファイル内に定義されていた `getClient` および `getUser` メソッドを削除してください。
   - `FetchTodos` 等の各メソッド内で、`r.getClient(ctx)` や `r.getUser(ctx)` と直接呼び出していた箇所を、`r.base.getClient(ctx)` や `r.base.getUser(ctx)` のように修正してください。

   **`backend/repositories/todo_filter_history.go` の修正:**
   - 同様に、構造体に `base *BaseRepository` を持たせ、`client *ent.Client` を削除してください。
   - コンストラクタ（`NewTodoFilterHistoryRepository`）を修正してください。
   - ファイル内に定義されていた `getClient` および `getUser` メソッドを削除してください。
   - `FetchLatestFilters` メソッド内での呼び出し箇所を `r.base.getClient(ctx)` や `r.base.getUser(ctx)` に修正してください。

3. **テストの実施**
   - リポジトリの内部構造の変更のみであるため、既存のサービス層やハンドラ層のテストがそのまま通るはずです。

4. **完了条件**
   - [ ] `cd backend && APP_ENV=ci go test ./...` ですべてのテストが成功すること。
   - [ ] ゴミや不要な変数・メソッド（未使用の `client` など）が残っていないか目視確認すること。
   - [ ] 「feat: extract getClient and getUser to BaseRepository in repositories」などの旨が分かる明確なメッセージでコミットを作成すること（このフェーズの変更内容のみを含めること）。

## サブエージェント向け共通ガイドライン

- `BaseRepository` メソッド名は、他パッケージから参照する必要がないため小文字始まり（非公開）の `getClient`, `getUser` のままでも機能するならそれで構いませんが、パッケージ内でどう扱うかは Go言語の規約とコンパイルエラーに沿って適切に定義してください。
- 常に1つの指示内容につき **1コミット** 作成すること。
- 直接関係しないファイル（`Orders/` ディレクトリ内のファイル等）はコミットに含めないよう注意してください。
