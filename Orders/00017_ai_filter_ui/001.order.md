## 背景

- ToDo 管理アプリのトップ画面では ToDo 一覧が表示されている
- 現在ユーザーがフィルタ可能な条件は「完了しているものを含む or 含まない」のみ
- 今回 AI を活用して、よりユーザーフレンドリーなフィルター UI を実装したい
  - Backend の API は既に実装済み
    - `GET /todo/ai_filter?query={query}`
      - 自由文で ToDo をフィルタする際に使う API。ToDo 一覧が返却される
      - レスポンスは `backend/dto/todo.go:8~15` で定義されている `TodoDto` の配列
    - `GET /todo/filter_histories`
      - 過去の検索クエリ履歴を取得する API
      - レスポンスは `backend/dto/todo.go:35~38` で定義されている `ListTodoFilterHistoriesResponseDto`
    - `GET /todo/filter_by_query_id?query_id={query_id}`
      　- 過去の検索クエリ ID (uuid) を指定し、ToDo 一覧を取得する API
      - レスポンスは `backend/dto/todo.go:8~15` で定義されている `TodoDto` の配列

## 要件

- トップ画面の ToDo 一覧画面に、AI フィルター機能を追加する
  - この機能を使ってフィルターが有効になっている間はページネーションは非表示でサーバーから返却された ToDo を全件表示する
- トップページをレンダリングする際、`GET /todo/filter_histories` を呼び出し、検索履歴一覧を取得しておく
- AI フィルター機能は、以下の UI で構成される
  - 検索バー
    - `Orders/00017_ai_filter_ui/Gemini_Generated_Image_q7rrnkq7rrnkq7rr.png` のような UI を実装する
    - クリックし、自由文を入力し検索ボタンを押す or Enter キーを押すと検索が実行される
      - 検索 API は `GET /todo/ai_filter?query={query}`
      - クエリは ToDo 一覧と検索バーの間に表示する
        - `Orders/00017_ai_filter_ui/Gemini_Generated_Image_w020l4w020l4w020.png` のような UI を実装する
        - バツボタンを押すと、検索履歴一覧と検索バーの間に表示しているクエリを削除し、通常の ToDo 一覧を表示する
    - レスポンスが返ってきたら現在表示している ToDo 一覧を、検索結果一覧に置き換える
      - 検索結果が 0 件の場合は ToDo が一件も表示されていない状態としてください。ユーザーは有効になっているクエリのバツボタンでフィルタを解除できます。
    - レスポンスが返ってくるまではローディングスピナーを表示する
  - 検索履歴一覧
    - 上記検索バーをクリックすると入力モードになるが、同時に検索履歴一覧をドロップダウン表示する
    - `Orders/00017_ai_filter_ui/Gemini_Generated_Image_7uo4697uo4697uo4.png` のような UI を実装する
      - トップページレンダリング時に取得した検索履歴一覧を表示する
    - 検索履歴一覧の各項目をクリックすると、そのクエリで検索が実行される
      - 検索 API は `GET /todo/filter_by_query_id?query_id={query_id}`
      - 選択した検索履歴のクエリは ToDo 一覧と検索バーの間に表示する
        - `Orders/00017_ai_filter_ui/Gemini_Generated_Image_w020l4w020l4w020.png` のような UI を実装する
        - バツボタンを押すと、検索履歴一覧と検索バーの間に表示しているクエリを削除し、通常の ToDo 一覧を表示する
