# 001.process_01: AI フィルター UI 実装の手順

サブエージェントは、以下の手順に沿って実装を進めてください。各フェーズが完了するごとに、対象ファイルのビルド確認やリンター実行などを通じてコードを検証し、**1フェーズ1コミット**でコミットを行ってください。
※フロントエンドのテストコードがないため、テスト実行は不要ですが型の整合性等には注意してください。

## Phase 2: カスタムフックの分離と拡張

`useTodos.ts` の肥大化を防ぐため、AIフィルター用のロジックは新しいフック `useAIFilter.ts` として切り出します。

### 2-1. `frontend/src/hooks/useAIFilter.ts` の新規作成

以下の要件を満たすフックを作成してください。

- **状態**:
  - `isFiltering` (boolean, 初期値: `false`)
  - `activeFilter` (`TodoFilterHistoryQuery | null`, 初期値: `null`)
  - `filterHistories` (`TodoFilterHistoryQuery[]`, 初期値: `[]`)
- **副作用 (useEffect)**:
  - マウント時に1度だけ `GET /todo/filter_histories` を呼び出し、レスポンスを `filterHistories` にセットする。
- **Zodバリデーション**:
  - ファイル上部で `const aiFilterSchema = z.string().trim().min(1, "入力してください").max(50, "50文字以内で入力してください");` を定義する。
  - スペースのみの文字列も `trim()` や `min(1)` によって弾かれるようにする。
- **引数**:
  - `onFilterSuccess: (todos: Todo[]) => void` (APIで取得したToDoを親（useTodosなど）に渡すためのコールバック)
  - `onClear: () => void` (フィルター解除時に親に通知するコールバック)
- **メソッド**:
  - `handleAIFilter(query: string, currentPage: number)`:
    1. `aiFilterSchema.safeParse(query)` でバリデーション。失敗時は `showToast` でエラーを表示し処理を中断（リクエストを発生させない）。
    2. `isFiltering` を `true` にする。
    3. `api.get("/todo/ai_filter", { params: { query: validation.data } })` を呼び出し、成功したら `onFilterSuccess(res.data)` を実行。
    4. `activeFilter` を `{ id: "custom", query: validation.data }` にセットする。
    5. 引数の `currentPage` が 1 ではない場合、`router.push("/?page=1")` を呼び出す。
    6. finally で `isFiltering` を `false` にする。
  - `handleHistoryFilter(history: TodoFilterHistoryQuery, currentPage: number)`:
    1. `isFiltering` を `true` にし、`api.get("/todo/filter_by_query_id", { params: { query_id: history.id } })` を実行。
    2. 成功したら `onFilterSuccess(res.data)` を実行し、`activeFilter` を `history` にセット。
    3. 引数の `currentPage` が 1 ではない場合、`router.push("/?page=1")` を呼び出す。
    4. finally で `isFiltering` を `false` にする。
  - `clearFilter()`:
    1. `activeFilter` を `null` にする。
    2. `onClear()` を呼び出す。
- **返り値**:
  - `isFiltering, activeFilter, filterHistories, handleAIFilter, handleHistoryFilter, clearFilter` をオブジェクトで返す。

### 2-2. `frontend/src/hooks/useTodos.ts` の調整

取得した `todos` の上書きや再取得を可能にし、同時にフィルター適用中のページネーション用 fetch を止めるための調整です。

1. **引数の追加**: `export function useTodos(skipFetch: boolean = false)` のように、一覧の自動取得をスキップするフラグを受け取れるようにする。
2. **useEffect の修正**: 引数 `skipFetch` が `true` の場合は、通常の `GET /todo` （ページネーション付き）のフェッチを実行せずリターンするように `useEffect` に条件分岐を追加。依存配列にも `skipFetch` を含める。
3. **返り値の追加**: 外部から状態を上書きできるよう、`setTodos` と、手動での再取得用に関数化された `fetchTodos` を return オブジェクトに含める。

## Phase 3: `AIFilterBar` コンポーネントの作成

`frontend/src/components/todo/AIFilterBar.tsx` を新規作成します。

### 3-1. Propsの定義

ファイル先頭付近に Props を定義します。

```typescript
import { TodoFilterHistoryQuery } from "@/types";

export interface AIFilterBarProps {
  filterHistories: TodoFilterHistoryQuery[];
  activeFilter: TodoFilterHistoryQuery | null;
  onSearchAIFilter: (query: string) => void;
  onSearchHistory: (history: TodoFilterHistoryQuery) => void;
  onClearFilter: () => void;
}
```

### 3-2. コンポーネント設計（デザイン要件）

以下の要素を用いて構成してください。指定したデザインの意図（画像からの推推測される構成）に沿って作成します。

- **アイコンの導入**:
  - `lucide-react` から `Search`, `Sparkles`, `X`, `History` アイコンを使用します。
- **検索バーのUIレイアウト**:
  - 外枠として、グラデーションの境界を持つコンテナなど、少し特別感のあるスタイルを適用します。（例: `relative p-[1px] bg-gradient-to-r from-purple-500 to-blue-500 rounded-lg` のようなラッパーを使用するか、シンプルに border カラーをブランド色に寄せるなど）。
  - 入力フィールド (`<Input>`) にはプレースホルダーとして `AIに頼んでタスクを絞り込む... (例: 「直近一週間以内に完了になったもの」)` を指定します（`maxLength={50}` 等も設定）。
  - 左側に `Sparkles` アイコンを青色 (`text-blue-500`) で絶対配置（`absolute left-3 top-1/2 -translate-y-1/2`）し、AIを使っているような特別感を演出します。テキストがかぶらないように `pl-10` などの padding を設定します。
  - 右側には `Search` アイコンを使用した検索ボタン（またはアイコン自体をクリッカブルに配置）を青色 (`text-blue-500`) で配置し、クリックで検索を実行させます。（`absolute right-3 top-1/2 -translate-y-1/2` など）
  - 全体を内包する要素で `relative` を用い、入力内容はコンポーネント内のローカル state (`inputValue`) に保持し、Enterキー押下、または検索ボタンクリックで `onSearchAIFilter(inputValue)` を実行させます。
- **検索履歴表示 (`Popover`)**:
  - 入力フィールド部分を構成するラッパー（または `<Input>` 自身）を `<PopoverTrigger asChild>` で囲います。
  - `<PopoverContent>` の中に、`filterHistories` をリストで表示します。
  - 各項目は、左側に時計アイコン（`History` from lucide-react）、右側にクエリ文字列という構成にし、`cursor-pointer hover:bg-muted p-2 text-sm rounded-md flex items-center gap-2` のようにスタイリングします。
  - 履歴をクリックした際、`onSearchHistory` を呼び出し、同時に Popover を閉じる state 管理（`open`, `onOpenChange` 等）を導入してください。
- **アクティブクエリの表示 (`Badge`)**:
  - `activeFilter` が `null` でない場合、検索バーの下に `<Badge variant="secondary" className="mt-4 px-3 py-1 flex w-fit items-center gap-2">` 等を用いて表示します。
  - Badge の左側には青色の `Sparkles` アイコン (`text-blue-500 className="w-4 h-4"`) を配置し、テキスト部分に `AI抽出: {activeFilter.query}` という形式で表示します。
  - バッジの右端には `<X className="w-4 h-4 cursor-pointer hover:text-destructive" onClick={onClearFilter} />` を配置してバツボタンとして機能させます。

## Phase 4: ホーム画面 (`page.tsx`) への統合

`frontend/src/app/page.tsx` を編集し、新規コンポーネントを組み込みます。

### 4-1. フックの統合と `AIFilterBar` の配置

1. `HomeContent` の中で、`useAIFilter` と `useTodos` を連携させます。
2. まず `useAIFilter` を呼び出し、状態（`activeFilter` など）を取得します。
   - `onFilterSuccess` コールバックには `setTodos` を渡してリストを更新させます。
   - `onClear` コールバックには `fetchTodos` を渡して通常の機能を再開させます。
3. `useTodos` には `skipFetch: activeFilter !== null` を渡し、フィルター中のみ通常の fetch を止めるよう設定します。
4. `AIFilterBar` を「完了したものを非表示にする」チェックボックス領域の下にマウントし、`useAIFilter` の状態とハンドラを渡します。ハンドラには `currentPage` 等必要な引数を渡すよう適宜ラッパー関数を作成してください。

### 4-2. スピナー表示とページネーション制御

- **ローディング:** `isFiltering` が true の間は、既存スピナー（`isLoading` に使われているもの）と同様のスピナーをリストの代わりに表示します。
- **ページネーション:** `activeFilter !== null` のときは（全体表示なので）、下部の `<PaginationControl>` を表示しないように条件分岐（`&& !activeFilter` 等）を追加します。

### 4-3. 検索結果ゼロ件の対応と解除

- 検索によって ToDo のレスポンスが 0 件だった場合は「ToDoが1件も表示されていない空の状態」としてください（既存の `todos.length === 0` の表示を活かして表示するなど）。
- 0件であっても、有効になっているクエリのバッジは表示され続けるため、ユーザーはそのバッジのバツボタンを押してフィルターを解除し、元の表示に戻せるように設計の意図を汲んで実装してください。

**コミットメッセージ:** `feat: integrate AI filter UI to home page`
