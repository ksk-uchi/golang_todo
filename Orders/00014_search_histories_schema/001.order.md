# 自由文入力による検索機能の検索履歴格納テーブル作成

## 背景

- ToDo 一覧を表示するアプリケーションとしてユーザーに自由文によるデータ検索機能を提供したい
  - 検索のたびに AI に対しトークンを消費するのを緩和する目的で、検索履歴を保存し再検索時に利用できるようにしたい
  - また、トークン消費対策だけでなく、クエリに対し正しい関数が呼び出されているか等、今後 AI にリクエストを送る際のプロンプトチューニングにも役立てたい

## 要件

- `todo_filter_histories` テーブルを作成する

## 対応内容

1. `backend/ent/schema/` に `todo_filter_histories` テーブルのスキーマを作成する
   - Columns
     - uuid (uuid)
       - uuid v7 が格納される予定 (可能であればサロゲートキーは使わずこのカラムを主キーにしたい)
     - user_id (int) (foreign key `users`)
     - query (varchar)
       - ユーザーが入力した自由文が格納される想定。400文字まで許容
     - function_name (varchar) (nullalbe)
       - どの機能を使ったかが格納される想定。100文字まで許容
     - args (json) (nullable)
       - function_name に渡した引数が格納される想定。
     - result_todo_ids (json) (nullable)
       - function_name の結果として返却された todo_ids が格納される想定。
     - created_at (datetime)
   - 完了条件
     - [ ] 全テストがパスすること
     - [ ] Git コミットの作成

## 制約事項

- Ent のスキーマ定義は `backend/ent/schema/` ディレクトリ配下に配置すること
