# 001.process_1 (todo_filter_histories スキーマ作成)

`Orders/00014_search_histories_schema/001.order.md` の要件を満たすための作業手順です。本手順書はサブエージェントが実行することを想定しています。

## 概要

`todo_filter_histories` テーブルに対応する Ent スキーマを `backend/ent/schema/` に `TodoFilterHistory` という名前で作成します。また、`User` スキーマとリレーションを持たせ、テスト実行によって変更に問題がないことを確認し、指定された単位でコミットを作成します。

## 作業手順

### 手順 1: TodoFilterHistory スキーマの生成と実装

1. `backend` ディレクトリへ移動し、`mise run new-schema TodoFilterHistory` を実行してスキーマファイルの雛形を作成します。
2. 生成された `backend/ent/schema/todofilterhistory.go` を開き、以下の要件を満たすように実装してください。

**todofilterhistory.go の実装要件:**

- パッケージは `package schema` とし、以下のモジュールをインポートします
  - `"time"`
  - `"entgo.io/ent"`
  - `"entgo.io/ent/dialect/entsql"`
  - `"entgo.io/ent/schema"`
  - `"entgo.io/ent/schema/edge"`
  - `"entgo.io/ent/schema/field"`
  - `"github.com/google/uuid"`
- `Annotations` メソッドを実装し、テーブル名を明示します (必要に応じて)。
  - `entsql.Annotation{Table: "todo_filter_histories"}`
- `Fields` メソッドを実装し、以下のカラムを定義します:
  - `id` (UUID): サロゲートキーを避けて主キーにするため、Ent におけるデフォルト主キーフィールド `id` の型を UUID に変更し、V7 でデフォルト生成されるようにします。
    - `field.UUID("id", uuid.UUID{}).Default(uuid.NewV7)`
  - `user_id` (int): `field.Int("user_id")`
  - `query` (varchar): `field.String("query").MaxLen(400)`
  - `function_name` (varchar nullable): `field.String("function_name").MaxLen(100).Optional()`
  - `args` (json nullable): 引数格納用 (オブジェクトを想定し map[string]interface{})。`field.JSON("args", map[string]interface{}{}).Optional()`
  - `result_todo_ids` (json nullable): TODO ID の配列格納用。`field.JSON("result_todo_ids", []int{}).Optional()`
  - `created_at` (datetime): `field.Time("created_at").Default(time.Now).Immutable()`
- `Edges` メソッドを実装し、`User` とのリレーションを定義します:
  - `edge.From("user", User.Type).Ref("todo_filter_histories").Unique().Field("user_id").Required()`

※注: Orderファイル上で uuid というカラム名にするよう指定がありますが、Entでは主キーフィールドは通常 `id` という名前になるため、主キーフィールドを `id` としつつ型を UUID に変更するアプローチをとっています。

### 手順 2: User スキーマの更新

1. `backend/ent/schema/user.go` を開き、`Edges()` メソッド内に `TodoFilterHistory` テーブルへのリレーションを追加します。
   - `edge.To("todo_filter_histories", TodoFilterHistory.Type).Annotations(entsql.OnDelete(entsql.Cascade))`

### 手順 3: Ent コードの生成とテスト実行

1. `backend` ディレクトリで `mise run ent-gen` (または `go generate ./ent`) を実行し、Ent のコードを生成します。
2. 生成が成功したら、`backend` ディレクトリ内で `go test ./...` を実行し、すべてのテストがパスすること（デグレードがないこと）を確認します。
   - テストエラー等が発生した場合は直ちに原因を特定・修正し、改めて全テストがパスすることを確認してください。

### 手順 4: コミットの作成

1. 対応内容の番号を振った段落単位でコミットを作成するよう指示されているため、ここまでのスキーマ追加・関連変更のみをステージングします。

   - `Orders` ディレクトリなど作業に無関係なファイルが含まれないよう、`git status` にて必ず確認してください。
3. `git commit -m "feat: add todo_filter_histories schema using ent"` などのメッセージでコミットを作成します。
