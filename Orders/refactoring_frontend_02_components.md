# ガイドライン: フロントエンド UI・Todoコンポーネント リファクタリング

対象の担当者: Frontend-Todo-Agent

※ `pnpm` を厳守してください（`npm`, `yarn`, `npx` 使用禁止）

## 目的

フロントエンドのメイン画面（Todo一覧）のロジック分離と、UIコンポーネントのファイル構成の整理を行います。
他のエージェントと並行稼働するため、指定された【対象ファイル】以外は極力変更しないでください。

## 対象ファイル

- `frontend/src/app/page.tsx`
- `frontend/src/app/layout.tsx`
- `frontend/src/app/components/*` (TodoItem.tsx, TodoModal.tsx 等)

## やるべきこと

1. **ロジックの分離 (Custom Hook)**:
   - メインの `page.tsx` において、状態管理・API非同期呼び出し・ページネーションなどが一箇所（約200行）に集中しています。これらを `hooks/useTodos.ts` のようなカスタムフックファイルを新規作成して分離・凝集させてください。
2. **UIコンポーネントのディレクトリ再構築**:
   - 現在 `components/ui/` に Shadcn パッケージの汎用UIと、ドメイン固有コンポーネント (`TodoItem`, `TodoModal`) が混在しています。これらを整理するため、例えば `components/todo/` などの明確なディレクトリにドメインコンポーネントを移動させ、全画面でのインポートパスを修正してください。
3. **TodoModal 等のZodバリデーション改善**:
   - `TodoModal` 内の手動による各種フォームバリデーションを `zod` スキーマを用いたバリデーションに置き換えてください。
   - `toastHandler` 等の個別定義や `suppressHydrationWarning` 等の不明瞭な対応を調査し、より適切な設定や `layout.tsx` での一元管理に移行してください。
4. **ビルドとテストの通過（必須要件）**:
   - 実装作業後、`pnpm build` を実行し、**エラーなくビルドできる事を確認**してから作業を完了してください。
   - **注意: 絶対に `git commit` や `git add` 等のgit操作は実行しないでください。** 修正のコミットや `pre-commit` フックの実行はすべてのエージェントの作業完了後にメインエージェントがまとめて行います。コードの修正とテスト確認のみを行って終了してください。
