# ToDo AI フィルターと履歴機能の実装手順

本ドキュメントは、メインエージェントによって作成された作業手順書です。
サブエージェントは以下の手順に沿って実装を行い、各フェーズごとにテストとコミットを実行してください。
途中でテストが失敗した場合は、修正をしてから次のフェーズへ進んでください。

## 前提条件

- データベースのスキーマやマイグレーション処理は済んでいる前提です。
- 各種モックは `testify/mock` を使用しています。
- DI コンテナは `Google Wire` を使用しています。シグネチャを変更した際は事前に必ず `wire_gen.go` の再生成を行ってください。

---

   - 実装内容: `r.base.getUser(ctx)` で取得したユーザー情報 (user_id) を紐づけて保存します。

2. **`backend/services/todo_filter_history.go` の修正**:

4. **テストの追加**:
   - `backend/services/todo_filter_history_test.go` に `SaveFilterHistory` に対する単体テストを定義してください。

### 完了条件 (Phase 1)

## Phase 2: `GET /todo/ai_filter` 用のハンドラーとルーティングの実装

### 概要

ユーザーからの自由文を受け取り、AIで結果をフィルタして履歴を保存し、その結果を返すエンドポイントを実装します。

### 作業内容

1. **依存関係の解決 (`backend/handlers/todo.go`)**:
   - `TodoHandler` 構造体および `NewTodoHandler` 関数のシグネチャを修正し、新たに `aiService *services.AIService` を受け取るように追加してください。
   - これに伴い、`backend/di` ディレクトリにて `wire_gen.go` を生成してください。（`cd backend/di && mise exec -- wire`）

2. **ハンドラーの実装 (`backend/handlers/todo.go`)**:
   - **`FilterTodosByQuery(c *echo.Context) error`**
     - クエリパラメータ `query` (string) を取得します。
     - `h.aiService.DecideFilterTodosFunction(ctx, query)` を呼び出します。
     - もし戻り値のDTOがあり、関数呼び出しが指示されている場合は、その関数名と引数を用いて `h.aiService.FilterTodos(ctx, dto.FunctionName, dto.Args)` を実行し `[]*ent.Todo` を取得します。指示されていない場合は空配列とします。
     - 取得した ToDo の ID の配列を作成し、`h.filterHistoryService.SaveFilterHistory` を呼び出して履歴を保存します。関数呼び出しがなかった場合は `functionName`, `args`, `result_todo_ids` に nil 等を適切に渡してください。
     - 得られた `[]*ent.Todo` を Dto に変換し、リストを JSON レスポンスとして返します（ステータス200）。

3. **ルーティングの追加 (`backend/routes/todo.go`)**:
   - `GET /todo/ai_filter` を `TodoHandler.FilterTodosByQuery` に紐づけてください。

4. **テストの追加 (`backend/handlers/todo_test.go`)**:
   - `FilterTodosByQuery` ハンドラーの正常系および異常系のテストケースを追加してください。

### 完了条件 (Phase 2)

- ビルドが通り、すべてのテストが成功すること。(`cd backend && go test ./...`)
- ここで2つ目のコミットをしてください。
  - コミットメッセージ例: `feat(backend): implement GET /todo/ai_filter endpoint`

## Phase 3: `GET /todo/filter_by_query_id` 用のリポジトリとサービス層の実装

### 概要

保存された履歴をIDベースで検索し、関連するToDoを展開するためのデータベース操作を実装します。

### 作業内容

1. **`backend/repositories/todo.go` の修正**:
   - `ITodoRepository` インターフェースに `FetchTodosByIds(ctx context.Context, ids []int) ([]*ent.Todo, error)` を追加し実装します。
   - 実装内容: `r.base.getUser(ctx)` で取得したユーザーに紐づき、かつ ID が `ids` に含まれる ToDo を取得（`Where(todo.IDIn(ids...))` などを使用）します。

2. **`backend/services/todo.go` の修正**:
   - `TodoService` に `FetchTodosByIds` を追加し実装します。

3. **`backend/repositories/todo_filter_history.go` の修正**:
   - `ITodoFilterHistoryRepository` に `GetFilterHistoryByQueryID(ctx context.Context, queryID uuid.UUID) (*ent.TodoFilterHistory, error)` メソッドを追加し実装します。
   - 実装内容: `r.base.getUser(ctx)` によるユーザー所有条件と `Where(todofilterhistory.IDEQ(queryID))` で履歴を取得します。

4. **`backend/services/todo_filter_history.go` の修正**:
   - `ITodoFilterHistoryService` に `GetFilterHistoryByQueryID` を追加し実装します。

5. **モックおよびテストの更新**:
   - `backend/testutils/mock_todo_repo.go` に `FetchTodosByIds` のモックを追加してください。
   - `backend/testutils/mock_todo_filter_history_repo.go` に `GetFilterHistoryByQueryID` のモックを追加してください。
   - それぞれのサービスメソッドに対して単体テストを追加してください。

### 完了条件 (Phase 3)

- 追加分の単体テストが成功すること。(`cd backend && go test ./...`)
- ここで3つ目のコミットをしてください。
  - コミットメッセージ例: `feat(backend): implement GetFilterHistoryByQueryID and FetchTodosByIds`

## Phase 4: `GET /todo/filter_by_query_id` 用のハンドラーとルーティングの実装

### 概要

履歴ID (uuid) を受け取り、結果として ToDo リストを返すエンドポイントを実装します。

### 作業内容

1. **ハンドラーの実装 (`backend/handlers/todo.go`)**:
   - **`FilterTodosByQueryID(c *echo.Context) error`**
     - クエリパラメータ `query_id` を取得し、`uuid.Parse` などで `uuid.UUID` に変換します。（エラー時は 400 BadRequest）
     - `h.filterHistoryService.GetFilterHistoryByQueryID(ctx, queryID)` を呼び出し、履歴データを取得します。
     - 履歴データが存在する場合、その `ResultTodoIds` を用いて `h.service.FetchTodosByIds(ctx, ids)` を呼び出します。
     - 得られた ToDo のリストを Dto に変換し、JSON レスポンスとして返します。

2. **ルーティングの追加 (`backend/routes/todo.go`)**:
   - `GET /todo/filter_by_query_id` を `TodoHandler.FilterTodosByQueryID` に紐づけてください。

3. **テストの追加 (`backend/handlers/todo_test.go`)**:
   - `FilterTodosByQueryID` 用の正常系やUUID不正フォーマット時などのテストケースを追加してください。

### 完了条件 (Phase 4)

- ビルドが通り、すべてのテストが成功すること。(`cd backend && go test ./...`)
- ここで4つ目の（最後の）コミットをしてください。
  - コミットメッセージ例: `feat(backend): implement GET /todo/filter_by_query_id endpoint`
