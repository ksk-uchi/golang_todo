# ToDo AI フィルターのクライアント初期化と DI 改善手順

本ドキュメントは、遅延初期化（Lazy Initialization）を伴う Factory パターンを実装するための作業手順書です。
サブエージェントは以下の手順に沿って実装を行い、各フェーズごとにテストとコミットを実行してください。
途中でコンパイルエラーやテストが失敗した場合は、修正をしてから次のフェーズへ進んでください。

## 前提条件

- `backend/services/ai.go` にあった「テスト用のグローバル変数を用いたモック化」というアンチパターンを取り除き、インターフェースに対する依存性の注入（DI）を用いた正しい設計に移行します。
- シグネチャを変更した際は事前に必ず `wire_gen.go` の再生成を行ってください。

---

## Phase 1: Factory パターンの作成 (`backend/utils/` 層)

1. **`backend/utils` ディレクトリの準備**
   - ない場合は新設してください。

2. **新規ファイル `backend/utils/ai_client_factory.go` の作成と実装**
   - 以下のインターフェースと構造体を定義します。戻り値として `services.IGenAIClient` （`todo-app/services` から import）を返す点に注意してください。
     - `IAIFactory` インターフェース (メソッド: `GetGeminiClient(ctx context.Context) (services.IGenAIClient, error)`)
     - `AIFactory` 構造体（フィールド: `geminiClient services.IGenAIClient`, `geminiOnce sync.Once`, `geminiErr error`）
   - コンストラクタ `NewAIFactory() IAIFactory` を作成し、`&AIFactory{}` を返すようにします。
   - `services/ai.go` に現在記述されている `genAIClientWrapper` 構造体および `GenerateContent` メソッドの実装をそっくりそのまま `backend/utils/ai_client_factory.go` に移動（カット＆ペースト）してください。
   - `GetGeminiClient` メソッドを実装します。`f.geminiOnce.Do(func() { ... })` の実行ブロック内で `genai.NewClient` を接続し、成功したら `&genAIClientWrapper{client: client}` を `f.geminiClient` に保存します。ブロックの外で `return f.geminiClient, f.geminiErr` とすることで、初回のみ生成・2回目以降はキャッシュされたクライアントを返す **Lazy Initialization** が実現されます。

### 完了条件 (Phase 1)

- 該当ファイル群のコンパイルが通り、ビルドエラーがないこと（`cd backend && go build ./utils/...` 等で確認）。

---

## Phase 2: `backend/services/ai.go` および単体テストのリファクタリング

1. **`backend/services/ai.go` の修正**:
   - `IGenAIClient` インターフェース定義は **そのまま残して** ください。
   - テスト用のためだけに存在していたグローバル変数 `createAIClientFunc`、ならびに `SetCreateAIClientFunc` および `ResetCreateAIClientFunc` 関数を **完全に削除** します。
   - （Phase 1 で移行した）`CreateAIClient` 関数および `genAIClientWrapper` の定義も削除します（すでに無い場合はスキップ）。
   - `DecideFilterTodosFunction` のシグネチャを以下のように更新します。
     `func (s *AIService) DecideFilterTodosFunction(ctx context.Context, aiClient IGenAIClient, query string) (*dto.AIFilterDto, error)`
   - 上記メソッド内で `createAIClientFunc` を呼び出していた箇所を削除し、引数で渡された `aiClient` を使って `aiClient.GenerateContent(...)` を呼び出すように変更します。

2. **`backend/services/ai_test.go` の更新**:
   - `createAIClientFunc` をオーバーライドしていた処理（`oldCreateAIClientFunc ...` や `createAIClientFunc = func...`）を削除します。
   - `s.DecideFilterTodosFunction` を呼び出す際、テスト内で定義している `mockClient` を直接引数に渡すように修正します（例： `s.DecideFilterTodosFunction(ctx, mockClient, "last month")`）。

### 完了条件 (Phase 2)

- 単体テストが成功すること。(`cd backend && go test ./services/...`)
- ここで1つ目のコミットをしてください。
  - コミットメッセージ例: `refactor(backend): move ai client to utils using factory pattern and cleanup global mocks in services`

---

## Phase 3: DI層 (Google Wire) のリファクタリング

1. **`backend/di/wire.go` の修正**:
   - `handlers.NewTodoHandler` が `utils.IAIFactory` に依存するようになるため、テスト時と本番時で Factory を適切に差し替えられるよう対応します。
   - `InitializeApp` 関数の `wire.Build(...)` ブロックの中に `utils.NewAIFactory,` を追加します。
   - `InitializeTestApp` 関数のシグネチャを変更して引数に直接モックを注入できるようにします。
     `func InitializeTestApp(e *echo.Echo, client *ent.Client, aiFactory utils.IAIFactory) (*App, error)`
     ※ 引数に指定することで、Wire はテスト時にそれをそのまま Provider（依存性）として認識して注入してくれます。

2. **`wire_gen.go` の再生成**:
   - コマンド `cd backend/di && mise exec -- wire` を実行し `wire_gen.go` を更新します。

### 完了条件 (Phase 3)

- `backend/di/wire_gen.go` が正常に生成されているか確認すること。（この時点では他所がコンパイルエラーになる可能性があります）

---

## Phase 4: ハンドラー層・テスト層の修正

1. **`backend/handlers/todo.go` の修正**:
   - `TodoHandler` 構造体に `aiFactory utils.IAIFactory` フィールドを追加します。
   - `NewTodoHandler` の引数に `aiFactory utils.IAIFactory` を追加し、構造体にセットします。
   - `FilterTodosByQuery` ハンドラー内で、AI 判定処理を呼ぶ前に `aiClient, err := h.aiFactory.GetGeminiClient(ctx)` を呼び出してクライアントを取得します。（エラー時は 500 InternalServerError 等で返すこと）
   - 取得した `aiClient` を引数に付与し、`h.aiService.DecideFilterTodosFunction(ctx, aiClient, query)` と呼び出すように修正します。

2. **`backend/handlers/todo_test.go` の修正**:
   - すべてのテストケースにある `di.InitializeTestApp(e, testClient)` の呼び出しを `di.InitializeTestApp(e, testClient, utils.NewAIFactory())` に変更してください（一括置換等で対応）。
   - ただし、AI モックが必要なテストの `TestTodoHandler_FilterTodosByQuery_Integration` については、以下の対応を行います。
     - `backend/handlers/todo_test.go` ファイル内で `utils.IAIFactory` を満たす `mockAIFactory` 構造体（`mock.Mock` 埋め込み）を定義し、`GetGeminiClient` をモック化します。
     - このテスト内で `mFactory := new(mockAIFactory)` を作成・セットアップし、`di.InitializeTestApp(e, testClient, mFactory)` のように**直接コンテナ構築処理へモックを引数で注入**します。これで完全に正しい DI トポロジーでのテストが完了します。
     - 以前の `services.SetCreateAIClientFunc` 関連のコードは削除してください。

### 完了条件 (Phase 4)

- ビルドが通り、すべてのテストが成功すること。(`cd backend && go test ./...`)
  （※データベースに依存するテストを実行させるために前回作成した `.env` ファイル等が必要であれば適宜作成・削除してください）
- ここで2つ目の（最後の）コミットをしてください。
  - コミットメッセージ例: `refactor(backend): inject IAIFactory via DI and update handlers/tests`

---

これらすべてのフェーズが完了したら作業完了となります。全体を通し、既存のコードスタイルや設計パターンに従ってコーディングを行ってください。
