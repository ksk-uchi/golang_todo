# WithTx 関数の配置の修正

本ドキュメントは、前回のリファクタリング過程でサブエージェントが一時的な Import Cycle (循環参照) エラーを回避するために `backend/services/todo.go` に移動させてしまった `WithTx` 関数を、本来あるべき `backend/utils/ent.go` に戻すための作業手順書です。

サブエージェントは以下の手順に沿って実装を行い、テストとコミットを実行してください。
途中でコンパイルエラーやテストが失敗した場合は、修正をしてから次のフェーズへ進んでください。

## 前提条件

- **※ 作業を始める前にターミナルで `backend` ディレクトリへ移動してください。すべてのパスやテスト実行はその階層を基準とします。**

---

## Phase 1: `WithTx` 関数の移動と呼び出し元の修正

1. **`backend/utils/ent.go` の再作成**:
   - 以下の内容で `backend/utils/ent.go` を作成（または再作成）してください。

     ```go
     package utils

     import (
     	"context"
     	"todo-app/ent"
     )

     // WithTx creates a new transaction and returns a new context with the transaction attached.
     // It uses ent.NewTxContext to attach the transaction to the context.
     func WithTx(ctx context.Context, client *ent.Client) (context.Context, *ent.Tx, error) {
     	tx, err := client.Tx(ctx)
     	if err != nil {
     		return nil, nil, err
     	}
     	// ent.NewTxContext attaches the transaction to the context,
     	// allowing ent.TxFromContext to retrieve it.
     	return ent.NewTxContext(ctx, tx), tx, nil
     }
     ```

2. **`backend/services/todo.go` の修正**:
   - 先頭の `import` ブロックに `"todo-app/utils"` を追加してください。
   - `WithTx` 関数の定義 (コメントを含めた func WithTx ... 全体) を削除してください。
   - `UpdateTodo` メソッド内の `WithTx(ctx, s.client)` の呼び出しを `utils.WithTx(ctx, s.client)` に変更してください。
   - `UpdateDoneStatus` メソッド内の `WithTx(ctx, s.client)` の呼び出しを `utils.WithTx(ctx, s.client)` に変更してください。

### 完了条件 (Phase 1)

- 既存の単体テストおよび統合テストがすべて成功すること (`cd backend && go test ./...`).
