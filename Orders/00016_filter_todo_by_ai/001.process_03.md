# 依存関係（Dependency）の修正と疎結合化

本ドキュメントは、前回実装で生じてしまった「`utils` 層から `services` 層への不適切な依存」というアーキテクチャ上の規約違反を解消するための作業手順書です。
Go言語の「暗黙的インターフェース」の特性を活かし、双方のパッケージが互いに import しない完全な疎結合を実現します。

サブエージェントは以下の手順に沿って実装を行い、各フェーズごとにテストとコミットを実行してください。
途中でコンパイルエラーやテストが失敗した場合は、修正をしてから次のフェーズへ進んでください。

## 前提条件

- 下位層である `utils` が上位層である `services` に依存してはならないというアーキテクチャ上の原則に従い、`backend/utils/ai_client_factory.go` から `todo-app/services` への依存を排除します。

---

## Phase 1: `utils` 層での独立したインターフェース定義

1. **`backend/utils/ai_client_factory.go` の修正**:
   - import ブロックに存在する `"todo-app/services"` を **削除** してください。
   - インターフェース `IGenAIClient` をこのファイル内に新しく定義してください。内容は以下の通りです。
     ```go
     type IGenAIClient interface {
     	GenerateContent(ctx context.Context, model string, contents []*genai.Content, config *genai.GenerateContentConfig) (*genai.GenerateContentResponse, error)
     }
     ```
   - 構造体 `AIFactory` の `geminiClient` フィールドの型を `services.IGenAIClient` から `IGenAIClient`（今定義したもの）に変更してください。
   - `IAIFactory` インターフェース内の `GetGeminiClient` の戻り値も `services.IGenAIClient` から `IGenAIClient` に変更してください。
   - 実装側である `func (f *AIFactory) GetGeminiClient(...)` の戻り値も `(IGenAIClient, error)` に変更してください。

### 完了条件 (Phase 1)

- `cd backend && go build ./utils/...` を実行し、`utils` パッケージ内でビルドエラーが出ないことを確認する。

---

## Phase 2: ハンドラー・テストの修正と統合

1. **`backend/handlers/todo_test.go` の修正**:
   - テストファイル内で前回定義した `mockAIFactory` 構造体のメソッド `GetGeminiClient` の戻り値が `(services.IGenAIClient, error)` となっているはずです。これを `(utils.IGenAIClient, error)` に修正してください。（これにより `utils` が返すインターフェースのシグネチャに型が合致します）
   - テスト内で作成した `mClient`（`mockGenAIClient`）はそのまま `utils.IGenAIClient` としても扱えるため、その他のテストロジックの修正は不要です。

2. **アーキテクチャの確認**:
   - `backend/services/ai.go` には依然として `services.IGenAIClient` が残っていますが、そこは**変更不要**です。
   - `backend/handlers/todo.go` では、生成した `aiClient` （`utils.IGenAIClient`）をそのまま `DecideFilterTodosFunction` （`services.IGenAIClient` を要求）に渡していますが、Go コンパイラが暗黙的に両者が同等であると判定するためエラーを出しません。

### 完了条件 (Phase 2)

- 全体のコンパイルとテストが成功すること。(`cd backend && go test ./...`)
- ここでコミットをしてください。
  - コミットメッセージ例: `refactor(backend): decouple utils from services using implicit interfaces`

---

すべてのフェーズが完了したら作業完了となります。
