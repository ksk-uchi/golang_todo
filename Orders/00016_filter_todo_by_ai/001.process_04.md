# 依存関係（Dependency）の修正とインターフェースの一元化

本ドキュメントは、Factory とインターフェースの配置を整理し、アーキテクチャをシンプルに再設計するための作業手順書です。

サブエージェントは以下の手順に沿って実装を行い、各フェーズごとにテストとコミットを実行してください。
途中でコンパイルエラーやテストが失敗した場合は、修正をしてから次のフェーズへ進んでください。

## 前提条件

- 「`utils` にインターフェースを定義し、`services` がそれをインポートして使う」という一方向のクリアな依存関係（レイヤードアーキテクチャ）に変更します。
- **※ 作業を始める前にターミナルで `backend` ディレクトリへ移動してください。すべてのパスやテスト実行はその階層を基準とします。**

---

## Phase 1: `utils` 層へのインターフェースの移動

1. **`backend/utils/ai_client_factory.go` の修正**:
   - `services.IGenAIClient` となっている全ての型宣言・戻り値指定を、自分自身（`utils` パッケージ内）で定義した `IGenAIClient` に変更します。
     - まず import ブロックに存在する `"todo-app/services"` を **完全に削除** してください。（`services` への依存の排除）
     - インターフェース `IGenAIClient` をファイルに新しく追加します。
       ```go
       type IGenAIClient interface {
       	GenerateContent(ctx context.Context, model string, contents []*genai.Content, config *genai.GenerateContentConfig) (*genai.GenerateContentResponse, error)
       }
       ```
     - 構造体 `AIFactory` の `geminiClient` フィールドを `IGenAIClient` に変更します。
     - `IAIFactory.GetGeminiClient` の戻り値、および実装メソッド `(f *AIFactory) GetGeminiClient` の戻り値を `(IGenAIClient, error)` に変更します。
   - ※これに伴い `genAIClientWrapper` は自動的に `utils.IGenAIClient` を満たすようになります。

---

## Phase 2: `services` 層とその他の修正

1. **`backend/services/ai.go` の修正**:
   - 元々このファイルにあった `type IGenAIClient interface { ... }` の定義を **完全に削除** します。
   - `DecideFilterTodosFunction` のシグネチャの引数 `aiClient IGenAIClient` を `aiClient utils.IGenAIClient` に変更します。
   - この変更のために、import ブロックに `"todo-app/utils"` を追加してください。

2. **`backend/services/ai_test.go` の修正**:
   - こちらテスト用で定義されている `MockGenAIClient` に対して `utils.IGenAIClient` と互換になるようにパッケージ名や参照が変わっていないか確認します（メソッドの実体自体は変わらないためそのままで動くか確認するのみです）。
   - ファイル内に古い `services.IGenAIClient` へのキャストなどがある場合は `utils` 側に向き先を変えてください。（必要に応じて `import "todo-app/utils"` を追加）

3. **`backend/handlers/todo_test.go` の修正**:
   - 定義されている `mockAIFactory` の `GetGeminiClient` メソッドのシグネチャおよび戻り値のキャストを以下のように変更します。
     - 戻り値: `(utils.IGenAIClient, error)`
     - キャスト部分: `return args.Get(0).(utils.IGenAIClient), args.Error(1)`
   - （元の実装で `import "todo-app/services"` が不要になっていれば削除します。）

### 完了条件 (Phase 1 & Phase 2 共通)

- 全体のコンパイルとテストが成功すること。(`cd backend && go test ./...`)
- テスト成功後、これらの変更を1つのコミットにまとめてください。
  - コミットメッセージ例: `refactor(backend): move IGenAIClient interface to utils to centralize AI client definitions`

---

すべてのフェーズが完了したら作業完了となります。
