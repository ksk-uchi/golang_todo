1. 以下の「要件」「背景」などを読み込み、不明点や仕様のエッジケースがあれば私（ユーザー）に質問してください。
2. 仕様が明確になったら、サブエージェントが迷わず実装・テスト・コミットできるように、具体的な指示を記した `001.process_01.md` を作成してください。（必要に応じて複数フェーズに分割し、01, 02... としてください）

## 背景

- ToDo 一覧画面において、ユーザーが自由文で ToDo をフィルタする機能のリリースを企画しています。

## 要件

- ユーザーが自由文で ToDo をフィルタする API を作成する

## 制約事項

- AI にクエリを投げて、結果から関数を呼び出し、フィルタされた ToDo を返すという実装はすでに `backend/services/ai.go` に `FilterTodos` というメソッドで実装してあります。そのため AI 呼び出しのためのロジック作成は不要です。

## 詳細仕様・実装ガイドライン（ユーザーが記入する場所）

（※ファイルパス、使用する関数名、関数のシグネチャ、責務の境界など、詳細に指定したいことがあれば以下に記載してください。メインエージェントは必ずこの指定を厳守し、サブエージェントへの指示書に反映させてください。）

- **対象ファイル・ディレクトリ:**
  - `backend/services/todo_filter_history.go`, `backend/repositories/todo_filter_history.go`
    - クエリとフィルタリング結果を保存するためのメソッド `SaveFilterHistory` を定義する
    - uuid を指定してフィルタ結果を取得するためのメソッド `GetFilterHistoryByQueryID` を定義する
      - user に紐づく ID であることを WHERE 句に含める
  - `backend/services/todo.go`, `backend/repositories/todo.go`
    - `FetchTodosByIds` メソッドを定義する
    - 上記変更にともなうテストの追記やモックの追記を行う
  - `backend/testutils/mock_todo_filter_history_repo.go`
    - `SaveFilterHistory` のモックを定義する
    - `GetFilterHistoryByQueryID` のモックを定義する
  - `backend/routes/todo.go`
    - `GET /todo/ai_filter?query=(string)` というエンドポイントを定義する
    - `GET /todo/filter_by_query_id?query_id=(uuid v7)` というエンドポイントを定義する
  - `backend/handlers/todo.go`
    - 上記のエンドポイントに対応するハンドラー `FilterTodosByQuery` と `FilterTodosByQueryID` を定義する
    - `FilterTodosByQuery` は下記の手順で動作する
      - `ai.go` の `DecideFilterTodosFunction` と `FilterTodos` を呼び出す
      - `backend/services/todo_filter_history.go` の `SaveFilterHistory` を呼び出す
        - `query`, `function_name`, `args`, `result_todo_ids` 等を保存する
        - もしクエリによるフィルター関数呼び出しが発生しなかった場合は `function_name`, `args`, `result_todo_ids` は NULL で `query` を保存する
      - `[]*ent.Todo` を返す
    - `FilterTodosByQueryID` は下記の手順で動作する
      - `backend/services/todo_filter_history.go` の `GetFilterHistoryByQueryID` を呼び出す
      - ToDo の ID リストが手に入るので `backend/services/todo.go` の `FetchTodosByIds` を使って `[]*ent.Todo` を作成し、返す
  - `backend/handlers/todo_test.go`
    - `FilterTodosByQuery` のテストケースを定義する
  - `backend/services/todo_filter_history_test.go`
    - `SaveFilterHistory` のテストケースを定義する
- **明確な責務の境界（やってほしくないこと等）:**
  - AI 呼び出しのためのロジック作成は不要

## サブエージェント向け手順書 (`process` ファイル) 作成時のガイドライン

- **ユーザーの「詳細仕様・実装ガイドライン」の意図を完全に読み取り、指定された関数名やファイルパスから逸脱しないこと。**
  - 不用意にファイルを分割したり、大規模なリファクタリングを指示したりしないでください。
- サブエージェントが「思考」や「設計」を必要としないレベルまで、修正対象ファイル・修正内容・追加するテストケースを具体化して記述すること。
- 作業をフェーズごとに分け、1フェーズが終わるごとに `go test ./...` 等によるテスト実行とコミット（1フェーズ1コミット）を指示すること。
- 独自に実装するよりも近しい機能を持つ実装がプロジェクトや標準ライブラリに存在する場合は、それを活用するよう指示すること。
